# 数字選択くじ 統計分析ラボ (numbers-lottery-lab)

日本の数字選択くじ（ロト7 / ロト6 / ミニロト / ナンバーズ3 / ナンバーズ4）の
**過去の当選データを取り込み、統計的な傾向を可視化して数字選びを楽しむ**ための
React 製ツールです。**エンタメ・学習目的**であり、当選を保証するものではありません。

> ⚠️ **重要**: 数字選択くじは毎回独立した乱数抽選です。過去の結果から次回を当てることは
> **数学的にできません**（どの数字も毎回、等確率で出ます）。本ツールの「予測」は傾向に沿った
> 数字生成にすぎず、当選確率を上げるものではありません。

## 特長

- **数字選択くじ全般に対応（設定駆動）** — `src/games/registry.js` に1エントリ追加するだけで
  新しいくじに横展開できる設計。
  - 組合せ型: ロト7（1–37/7個）、ロト6（1–43/6個）、ミニロト（1–31/5個）
  - 桁型: ナンバーズ3（3桁）、ナンバーズ4（4桁）
- **統計の可視化** — 出現頻度ヒートマップ、高頻度/ご無沙汰ランキング、奇数・低位・合計値の平均、
  共起ペア（組合せ型）、桁×数字ヒートマップ（桁型）。
- **3つの生成戦略** — 均等ランダム（数学的に最も公正）／高頻度バイアス／ご無沙汰バイアス。
- **3系統のデータ取込** — テキスト貼り付け／CSVファイル（みずほ銀行公式の当せん番号CSV、
  Shift_JIS・UTF-8 自動判定、ドラッグ＆ドロップ対応）／スクリーンショット画像のOCR読み取り。
- **二重取込の防止** — 回号（第○○回）をキーに取込済みの回は自動スキップ。同じ回号で数字が
  異なる場合のみ「上書きしますか？」と確認。結果は「新規○件・スキップ○件・上書き○件」で表示し、
  「取込済み: 第○回〜第○回（計○件）」を常時表示。
- **取込済みデータの管理画面** — 全データを回号順に一覧（スクロール）表示し、各行の編集
  （くじ仕様でバリデーション）・削除、全削除（サンプルに戻す）が可能。
- **PWA対応（オフライン動作・ホーム画面追加）** — Service Worker でアプリ本体を
  プリキャッシュし、機内モードでも完全動作。スマホでは「ホーム画面に追加」で
  ネイティブアプリ風（全画面・アイコン起動）に使える。フォントもローカル同梱で
  **実行時の外部通信は完全にゼロ**。OCRアセット（約9MB）は初回利用時にキャッシュされ、
  一度使えば以後オフラインでもOCRが動く。

## 設計方針（法的・セキュリティ面）

このアプリは STEPN 等の他アプリとは独立した単体プロジェクトです。以下を前提に設計しています。

- **当選を保証しない旨を UI に明示** — 景品表示法上の優良誤認リスクを避けるため。
- **外部通信ゼロ** — 公式サイトのスクレイピング（利用規約・サーバ負荷の懸念）や当選番号
  データベースの再配布（データベース権の懸念）を避け、**ユーザー自身がデータを取り込む方式**を採用。
  初回は「架空のサンプルデータ」で動作します（実在の当選番号ではありません）。
  OCR（Tesseract.js）も既定のCDN取得を使わず、worker・wasmコア・日本語言語データを
  `public/tesseract/` に**ローカル同梱**して実行時の外部通信を発生させません
  （`scripts/copy-tesseract-assets.js` が `npm install` 時に node_modules から自動コピー）。
  画像もテキストもすべて端末内で処理されます。
- **完全クライアントサイド** — 個人情報の収集・サーバ送信は一切なし。データはブラウザの
  LocalStorage にゲームごとに分離して保存し、入力は各くじの仕様に沿って厳格にバリデーションします。
- **責任あるギャンブル** — 20歳以上・自己責任・無理のない範囲での利用を案内。

## データ取り込み

「💾 過去データ取込・管理」パネルから3つの方法で取り込めます。いずれも回号キーで
重複を自動スキップし、数字が異なる場合のみ上書き確認します。

### 1. テキスト貼り付け

1行1回ぶん、カンマ/空白区切り。

```
# 組合せ型（ロト7の例）: 回号, 日付, 本数字×7, ボーナス×2
500, 2023-01-06, 3, 9, 12, 18, 25, 30, 36, 7, 21

# 桁型（ナンバーズ4の例）: 回号, 日付, 当選番号
1000, 2023-01-06, 1234
```

### 2. CSVファイル

みずほ銀行公式サイトで配布されている過去当せん番号CSVを、ファイル選択または
ドラッグ＆ドロップで一括取込。文字コードは Shift_JIS / UTF-8 を自動判定し、
「第○回」「和暦日付」「(ボーナス数字)」「賞金額などの付随列」といった列構成の揺れを
ヒューリスティックに吸収します（全ゲーム対応）。

### 3. スクリーンショットOCR

当せん結果一覧のスクショ画像を Tesseract.js（完全ローカル同梱）で読み取ります。
誤認識に備えて、読み取り結果は必ず**確認・修正画面**に表示され、読み取れなかった行も
明示されます。ユーザーが修正・確定してはじめて保存されます。

## 対象外（将来フェーズ）

toto / BIG や競馬・競輪・競艇などの「結果予想型」は、乱数抽選ではなく実際の試合・レース結果に
依存するため、**別エンジンと実データが必要**で、データ取得のルール順守や控除率（期待値マイナス）の
明示といった課題が大きく、本リポジトリのスコープ外としています。

## 開発

```bash
npm install
npm start          # 開発サーバ
npm test           # テスト（エンジンの単体テスト含む）
npm run build      # 本番ビルド
```

Create React App ベース。主要コード:
- `src/games/registry.js` — くじ仕様
- `src/engine/combination.js`, `src/engine/digits.js` — 統計・生成エンジン
- `src/views/CombinationView.js`, `src/views/DigitsView.js` — 表示
- `src/App.js` — くじ種セレクター＋シェル
- `src/components/DataPanel.js` — データ取込（テキスト/CSV/OCR）＋一覧・編集・削除
- `src/lib/csv.js` — CSV/OCRテキストの寛容パーサ（Shift_JIS判定含む）
- `src/lib/merge.js` — 回号キーの二重取込防止マージ
- `src/lib/validate.js` — くじ仕様バリデーション（編集フォーム用）
- `src/lib/ocr.js` — Tesseract.js ラッパー（ローカル同梱アセット参照）
- `scripts/copy-tesseract-assets.js` — OCRアセットの `public/tesseract/` への同梱
- `src/service-worker.js`, `src/serviceWorkerRegistration.js` — PWA（オフライン対応）。
  本体はプリキャッシュ、OCRアセットは初回利用時に CacheFirst でキャッシュ。
  Service Worker は本番ビルド（`npm run build`）でのみ有効。

将来ストア配布（App Store / Google Play）したくなった場合は、このビルドを
Capacitor でラップするのが最短ルート（コードはそのまま流用可能）。
